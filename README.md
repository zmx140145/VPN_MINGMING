# VPN 配置说明

## 配置文件使用指南

### 文件说明
- `myVpn.yaml` - Clash 代理配置文件

### 使用方法

#### 1. 保存文件
将 `myVpn.yaml` 文件保存到你的电脑或手机上。

#### 2. 导入 Clash 客户端

**Clash for Windows：**
- 直接将文件拖拽到客户端的 Profiles 页面

**Clash for Android：**
- 将文件传入手机存储
- 在 App 内通过 `配置 → 新建 → 从文件导入` 选择文件

**Clash for Mac：**
- 打开 ClashX，点击菜单栏图标
- 选择 `配置 → 打开配置文件夹`
- 将 `myVpn.yaml` 复制到配置文件夹
- 在 `配置` 菜单中选择该配置文件

**Clash for iOS：**
- 使用支持文件导入的 Clash 客户端（如 Shadowrocket）
- 通过文件共享或 AirDrop 传输配置文件

#### 3. 作为订阅链接使用（可选）
你可以将此文件上传到 GitHub Gist、服务器目录，或任何能通过直链访问的地方，然后在客户端通过 URL 方式导入，实现动态更新。

**GitHub Gist 方式：**
1. 访问 https://gist.github.com
2. 创建新的 Gist，将配置文件内容粘贴进去
3. 点击 "Create public gist"
4. 点击 "Raw" 获取直链
5. 在 Clash 客户端中使用该 URL 作为订阅链接

**GitHub 仓库方式（推荐）：**
1. 访问此仓库的 `myVpn.yaml` 文件
2. 点击 "Raw" 按钮获取直链
3. 在 Clash 客户端中使用该 URL 作为订阅链接

#### 4. 启动连接
1. 在客户端的 **代理（Proxies）** 页面
2. 选择 **PROXY** 组
3. 点选 **美国服务器** 节点
4. 打开系统代理开关

### 配置说明

- **端口设置：**
  - HTTP 代理端口：7890
  - SOCKS5 代理端口：7891
  - 外部控制器：0.0.0.0:9090

- **代理节点：**
  - 名称：美国服务器
  - 类型：VMESS
  - 服务器：154.12.36.176
  - 端口：443

- **规则说明：**
  - Google、GitHub、YouTube 等网站走代理
  - 中国大陆 IP 直连
  - 其他流量默认走代理

### 注意事项

⚠️ **重要提示：**
- 请妥善保管配置文件，避免泄露服务器信息
- 定期检查服务器连接状态
- 如遇到连接问题，请检查服务器地址和端口是否正确
